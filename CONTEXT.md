## Product goal

Single-click Windows product so a non-technical user can install one app, open one desktop shortcut, click Start All, and let an MCP-compatible LLM safely control Chrome (DOM-first) with computer-use fallback and full permission gates.

## Current scope (MVP vertical slice)

1. Control Center shell (Electron)
2. Process supervisor (Start All / Stop All / Emergency Stop)
3. Local MCP server (stdio + optional loopback HTTP)
4. Chrome MV3 extension + Native Messaging bridge
5. Browser tools: status, list tabs, read page, click, type, scroll, screenshot
6. Permission engine (levels 0–3)
7. Audit log + redaction
8. LLM pairing (Grok CLI, Claude, Codex, generic)
9. Health check + Repair System
10. Documentation + tests

**Deferred (post vertical-slice pass):** advanced autofill UI polish, full OCR computer-use, signed code-signing cert, workflow recorder UI, Web Store publish.

## Architecture (summary)

Control Center (Electron) → Supervisor → MCP runtime + Native Host IPC → Chrome Extension → Tabs/DOM.  
All tools pass PermissionEngine. Emergency stop blocks new actions.

## Assumptions

- Windows 10/11 x64
- Google Chrome installed
- Node 20+ for development; packaged app embeds runtime via Electron
- Extension may run unpacked during development (ID generated by Chrome)
- Code signing certificate may be absent (unsigned installer noted as risk)

## Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | TypeScript + MCP SDK | Typed tools, ecosystem standard |
| D2 | Electron Control Center | Single desktop app, no CLI for users |
| D3 | Native Messaging primary path | Uses existing Chrome session |
| D4 | stdio MCP default | Grok CLI / Claude / Codex compatible |
| D5 | Loopback HTTP optional + token | Dashboard + local clients only |
| D6 | Permission levels 0–3; L3 blocked MVP | Zero silent high-risk actions |
| D7 | Vertical slice before autofill depth | Spec §20 |

## Constraints

- No public bind by default
- No secrets to LLM (passwords/OTP/payment)
- Never combine fill + submit
- No false Ready status
- Emergency Stop always available

## Security risks

| Risk | Severity | Mitigation |
|------|----------|------------|
| Unsigned installer | High | Document; support Authenticode when cert available |
| Unpacked extension ID volatility | Medium | Repair re-registers host; wizard guides |
| Prompt injection via page text | High | Treat page as untrusted data; structured tools only |
| Local MCP client spoofing | Medium | Pairing tokens; loopback + auth |
